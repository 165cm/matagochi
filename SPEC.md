# またごち MVP 仕様書

## 1. 概要

**またごち** は、家族の「また食べたい」を残す、ごはんメモアプリです。

SNSで見つけたレシピ動画を保存して、作ったら家族の星評価を記録します。「また作る？」がひと目でわかる、忙しい家庭のための軽いレシピ動画メモを目指します。

旧仮称は **KondaTETRiS** でした。公開にあたり、正式名称を **またごち** に変更します。

## 2. キャッチコピー

家族の「また食べたい」を残す、ごはんメモ

## 3. サービス説明

SNSで見つけたレシピ動画を保存して、作ったら家族の星評価を記録。

「また作る？」がひと目でわかる、忙しい家庭のごはんメモ。

## 4. 想定ユーザー

- SNSでレシピ動画を見つけるが、あとから探し直すことが多い人
- 家族の好みや「次いつ食べたいか」を残したい人
- レシピアプリほど重くない、家庭用のごはんメモが欲しい人
- 忙しい平日でも、作る候補を短時間で決めたい人

主な初期ターゲットは、毎日の献立に悩む働くママです。

## 5. MVP の目的

MVPでは以下の2つに集中します。

1. ショート動画レシピのコレクション
2. 作ったレシピの家族評価

献立表、買い物リスト、栄養アドバイスは将来拡張候補とし、まずは「保存する」「評価する」「また作る判断をする」という最小価値を検証します。

## 6. 画面構成

### 初回オンボーディング

データが未保存の初回起動時に表示します。「空ではじめる」か「サンプルを見る」を選んでから本編に入ります。選択後はlocalStorageに記録し、次回以降は表示しません。

### コレクション画面

ショート動画レシピを保存・検索する画面です。

主な要素:

- レシピ数、高評価候補、未評価数、材料メモ数のサマリー
- レシピ名入力
- ショート動画URL入力
- 対応プラットフォーム判定
- 動画元入力
- 朝ごはん、昼ごはん、夜ごはん、副菜の分類
- キャプション入力
- 自分用メモ入力
- URLから取得ボタン
- キャプションから材料・作り方を抽出ボタン
- 材料プレビュー
- 調理方法プレビュー
- 保存済みレシピ一覧
- レシピ検索
- 動画を開く
- 評価する
- レシピの編集（同じフォームで内容を更新）
- レシピの削除（確認ダイアログ付き、関連評価も連動削除）

### 評価画面

保存済みレシピを選び、家族の評価を記録する画面です。

主な要素:

- 評価件数
- 選択中レシピの平均評価
- 作った回数
- 早めに再登場候補の数
- 評価するレシピ選択
- レシピ概要
- 家族メンバーごとの星評価（メンバー数は設定で変更可能）
- 次はいつ頃食べたいか
- 評価メモ
- 料理写真の登録（任意）
- 評価履歴
- 評価履歴の削除

### 設定画面

家族メンバーとデータを管理する画面です。

主な要素:

- 家族メンバー数・レシピ数・評価数のサマリー
- 家族メンバーの追加・名称変更・削除
- データの書き出し（JSONバックアップ）
- データの読み込み（JSONバックアップからの復元）

## 7. 対応ショート動画サービス

現状はURL判定に対応しています。

- Instagram Reel
- Facebook Reel / Watch / Video
- TikTok 動画URL
- YouTube Shorts

現状できること:

- URLからサービス種別を判定する
- 動画元フィールドへ反映する
- 対応URLかどうかを表示する
- 保存済みレシピから動画URLを開く

現状できないこと:

- URLだけから各SNSのキャプション本文を自動取得する

理由:

- 静的フロントエンドだけではCORS制限がある
- Instagram、Facebook、TikTokはログインやAPI制限の影響を受ける
- YouTube Shortsも説明文取得にはAPIキーやバックエンド連携が必要

将来対応:

- `URLから取得` ボタンをバックエンド/API連携に差し替える
- サーバー側でURLを受け取り、許可された方法でメタ情報やキャプションを取得する
- 取得できない場合は、現在と同じくキャプション貼り付け運用へ戻す

## 8. データ仕様

データは現状 `localStorage` に保存します。
PCとスマホなど別端末で使う場合は、JSONファイルを書き出して読み込みます。

保存キー:

- `matagochi-mvp-v1`

### JSON Backup

設定画面から現在の保存データをJSONファイルとして書き出せます。
読み込み時は現在の保存データを、JSONの内容で置き換えます。

```json
{
  "view": "collection",
  "family": ["ママ", "パパ", "子ども1", "子ども2"],
  "recipes": [],
  "evaluations": [],
  "onboarded": true
}
```

### Recipe

```json
{
  "id": "r1",
  "title": "小松菜と卵の朝スープ",
  "videoUrl": "https://...",
  "source": "TikTok",
  "mealType": "breakfast",
  "caption": "小松菜 1束、卵 2個...",
  "ingredients": [
    { "name": "小松菜", "amount": "1束", "category": "野菜" }
  ],
  "steps": ["小松菜と豆腐を食べやすく切る"],
  "tags": ["朝ごはん", "TikTok", "動画"],
  "savedAt": "2026-06-21",
  "note": "朝ごはんの固定枠にできそう。"
}
```

### Evaluation

```json
{
  "id": "e1",
  "recipeId": "r1",
  "cookedAt": "2026-06-21",
  "familyRatings": {
    "ママ": 5,
    "パパ": 4,
    "子ども1": 4,
    "子ども2": 3
  },
  "nextTiming": "2週間後",
  "memo": "子ども用はしょうゆ少なめ。",
  "photo": "data:image/jpeg;base64,...",
  "photoLabel": "食卓写真"
}
```

`photo` は任意で、登録された料理写真をリサイズ後の data URL（JPEG）として保持します。
家族メンバーは `state.family`（文字列配列）で管理し、`familyRatings` のキーと対応します。メンバー名を変更すると、既存の評価とドラフトのキーも合わせて移行します。
初回オンボーディングの完了は `state.onboarded` に記録します。`savedAt` と `cookedAt` は保存・評価時の端末の現在日付を使用します（上記の例値はサンプルデータの固定値）。

## 9. 抽出ロジック

### 材料抽出

キャプションから以下のような表現を拾います。

- `鮭 3切れ`
- `しめじ 1袋`
- `玉ねぎ 1個`
- `しょうゆ 大さじ1`
- `めんつゆ 適量`

抽出後、材料カテゴリを簡易判定します。

- 野菜
- 肉
- 魚
- 卵・乳製品
- 大豆・加工品
- 主食
- 缶詰
- 調味料
- その他

### 調理方法抽出

キャプション内の文章から、調理に関係する動詞を含む文を手順として抽出します。

対象例:

- 切る
- 炒める
- 焼く
- 煮る
- 蒸す
- 混ぜる
- のせる
- かける
- 包む
- 加える
- ゆでる
- 冷やす
- 仕上げる

番号付き手順がある場合は、番号を外して最大6件まで表示します。

## 10. 現状の制約

- バックエンドなし
- ログインなし
- ユーザーアカウントなし
- 端末間の自動同期なし（JSON書き出し／読み込みによる手動移行は可能）
- 写真はサーバーへアップロードせず、リサイズして端末内（localStorage）に保存
- SNSキャプションの実取得なし
- データは端末ごとのブラウザ保存
- 保存・評価の日付は端末の現在日付を使用（既存デモデータの日付は固定）

GitHub Pagesで動くこと:

- アプリ画面表示
- URL入力
- 対応サービス判定
- キャプション貼り付け
- 材料・調理方法の抽出
- レシピ保存
- 評価保存
- localStorage保存
- JSONファイルの書き出しと読み込み

GitHub Pagesだけでは動かないこと:

- SNS URLからキャプション本文を自動取得すること
- ログイン付きの自動データ同期

## 11. 今後の拡張候補

実装済み:

- レシピ編集
- レシピ削除
- 評価履歴の削除
- 家族メンバー編集（追加・名称変更・削除）
- 保存データの書き出し／読み込み（JSONバックアップ）
- 実日付での記録
- 初回オンボーディング（空ではじめる／サンプルを見る）
- 削除・リセットの確認ダイアログ
- 入力バリデーションと保存後のフォームクリア
- 写真登録（評価への料理写真、端末内保存・自動リサイズ）

優先度高:

- URLキャプション取得用バックエンド

優先度中:

- 月間評価カレンダー
- お気に入り、作りたい、作ったタグ
- 買い物メモ生成
- 献立表への配置

優先度低:

- 栄養バランスアドバイス
- AIによる献立穴埋め提案
- 複数端末同期
- 共有家族アカウント

## 12. MVP 成功条件

- ユーザーがショート動画URLを保存できる
- キャプション貼り付けから材料と調理方法を確認できる
- 作った後に家族評価を残せる
- 保存済みレシピから「また作るか」を判断できる
- JSONで保存データを別端末へ持ち運べる
- GitHub Pagesなど静的ホスティングでも基本機能を試せる
