# リピごち MVP 仕様書

## 1. 概要

**リピごち** は、家族の「また食べたい周期」を残す、ごはんメモアプリです。

SNSで見つけたレシピ動画を保存し、作った後に家族ごとのリピート希望周期を記録します。記録した周期と直近の調理日から、今週食べ頃の献立候補を提案します。

公開URLとlocalStorageキーは既存ユーザー互換のため、旧名称時代の `matagochi` を維持します。

## 2. キャッチコピー

家族の「また食べたい周期」を残す、ごはんメモ

## 3. 想定ユーザー

- SNSでレシピ動画を見つけるが、あとから探し直すことが多い人
- 家族の「また食べたい」ペースを残したい人
- 忙しい週でも、作る候補を短時間で決めたい人
- レシピアプリほど重くない、家庭用のごはんメモが欲しい人

主な初期ターゲットは、毎日の献立に悩む働くママです。

## 4. MVP の目的

MVPでは以下の3つに集中します。

1. ショート動画レシピのコレクション
2. 作ったレシピの家族別リピ周期記録
3. リピ周期にもとづく1週間の献立提案

買い物リスト、栄養最適化、SNS本文の自動取得、クラウド同期は将来拡張候補です。

## 5. 画面構成

### 初回オンボーディング

データが未保存の初回起動時に表示します。「空ではじめる」か「サンプルを見る」を選んでから本編に入ります。選択後はlocalStorageに記録し、次回以降は表示しません。

### コレクション画面

ショート動画レシピを保存・検索する画面です。

主な要素:

- レシピ数、今週候補数、未リピ記録数、材料メモ数のサマリー
- レシピ名入力
- ショート動画URL入力
- 対応プラットフォーム判定
- 動画元入力
- 朝ごはん、昼ごはん、夜ごはん、副菜の分類
- キャプション入力
- 自分用メモ入力
- URLから取得ボタン
- キャプションから材料・作り方を抽出ボタン
- 材料プレビュー
- 調理方法プレビュー
- 保存済みレシピ一覧
- レシピ検索
- 動画を開く
- リピ記録
- レシピの編集
- レシピの削除（確認ダイアログ付き、関連リピ記録も連動削除）

### 献立画面

リピ周期から1週間分の献立候補を表示する画面です。

主な要素:

- 7日分の献立提案
- 今週食べ頃、周期未設定、リピなしのサマリー
- 食べ頃候補一覧
- 候補からリピ周期画面への導線
- 候補から元動画を開く導線

### リピ周期画面

保存済みレシピを選び、家族ごとのリピート希望周期を記録する画面です。

主な要素:

- リピ記録件数
- 選択中レシピの代表周期
- 作った回数
- 今週候補数
- 記録するレシピ選択
- レシピ概要
- 家族メンバーごとの6段階周期
- メモ
- 料理写真の登録（任意）
- リピ記録履歴
- リピ記録履歴の削除

### 設定画面

家族メンバーとデータを管理する画面です。

主な要素:

- 家族メンバー数・レシピ数・リピ記録数のサマリー
- 家族メンバーの追加・名称変更・削除
- データの書き出し（JSONバックアップ）
- データの読み込み（JSONバックアップからの復元）

## 6. リピ周期の6段階

| 表示 | 内部ID | 日数換算 | 献立提案での扱い |
|---|---|---:|---|
| 明日でも | `tomorrow` | 1 | 最優先で再登場候補にする |
| 毎週 | `weekly` | 7 | 週1ペースで提案する |
| 月２回 | `twice_month` | 14 | 約2週間ペースで提案する |
| 月１回 | `monthly` | 30 | 約1か月ペースで提案する |
| しばらくいい | `pause` | 90 | 低優先度にする |
| リピなし | `never` | なし | 献立提案から除外する |

家族全員が `リピなし` のレシピは、献立候補に出しません。一部の家族だけが `リピなし` の場合は、その人を平均周期の計算から外します。

## 7. 献立提案ロジック

1. レシピごとの最新リピ記録を取得する。
2. `リピなし` 以外の家族周期を日数へ変換する。
3. 有効な周期の平均値を、次の食べ頃日として直近調理日に加算する。
4. 食べ頃日が近い、または過ぎているレシピを優先する。
5. 1週間の中で同じレシピが重複しないように配置する。
6. 候補不足時は、未記録レシピを「まだ周期未設定」として候補に残す。

このMVPでは、曜日別の朝昼夜最適化、栄養バランス、買い物リスト生成は行いません。

## 8. 対応ショート動画サービス

現状はURL判定に対応しています。

- Instagram Reel
- Facebook Reel / Watch / Video
- TikTok 動画URL
- YouTube Shorts

現状できること:

- URLからサービス種別を判定する
- 動画元フィールドへ反映する
- 対応URLかどうかを表示する
- 保存済みレシピから動画URLを開く

現状できないこと:

- URLだけから各SNSのキャプション本文を自動取得する

理由:

- 静的フロントエンドだけではCORS制限がある
- Instagram、Facebook、TikTokはログインやAPI制限の影響を受ける
- YouTube Shortsも説明文取得にはAPIキーやバックエンド連携が必要

## 9. データ仕様

データは現状 `localStorage` に保存します。
PCとスマホなど別端末で使う場合は、JSONファイルを書き出して読み込みます。

保存キー:

- `matagochi-mvp-v1`

### JSON Backup

設定画面から現在の保存データをJSONファイルとして書き出せます。
読み込み時は現在の保存データを、JSONの内容で置き換えます。

互換性維持のため、配列名 `evaluations` は残します。新しい記録では `familyRatings` ではなく `familyRepeatCycles` を使います。

```json
{
  "view": "collection",
  "family": ["ママ", "パパ", "子ども1", "子ども2"],
  "recipes": [],
  "evaluations": [],
  "repeatDraft": {
    "familyRepeatCycles": {
      "ママ": "weekly",
      "パパ": "weekly"
    },
    "memo": "",
    "photo": ""
  },
  "onboarded": true
}
```

### Recipe

```json
{
  "id": "r1",
  "title": "小松菜と卵の朝スープ",
  "videoUrl": "https://...",
  "source": "TikTok",
  "mealType": "breakfast",
  "caption": "小松菜 1束、卵 2個...",
  "ingredients": [
    { "name": "小松菜", "amount": "1束", "category": "野菜" }
  ],
  "steps": ["小松菜と豆腐を食べやすく切る"],
  "tags": ["朝ごはん", "TikTok", "動画"],
  "savedAt": "2026-06-21",
  "note": "朝ごはんの固定枠にできそう。"
}
```

### Evaluation / Repeat Record

```json
{
  "id": "e1",
  "recipeId": "r1",
  "cookedAt": "2026-06-21",
  "familyRepeatCycles": {
    "ママ": "weekly",
    "パパ": "twice_month",
    "子ども1": "monthly",
    "子ども2": "never"
  },
  "memo": "子ども用はしょうゆ少なめ。",
  "photo": "data:image/jpeg;base64,...",
  "photoLabel": "食卓写真"
}
```

`photo` は任意で、登録された料理写真をリサイズ後の data URL（JPEG）として保持します。
家族メンバーは `state.family`（文字列配列）で管理し、`familyRepeatCycles` のキーと対応します。メンバー名を変更すると、既存のリピ記録とドラフトのキーも合わせて移行します。
初回オンボーディングの完了は `state.onboarded` に記録します。`savedAt` と `cookedAt` は保存・記録時の端末の現在日付を使用します（上記の例値はサンプルデータの固定値）。

### 旧データ互換

旧MVPの `familyRatings` は、読み込み時に以下のように `familyRepeatCycles` へ変換します。

| 旧星評価 | 新周期 |
|---:|---|
| 5 | 毎週 |
| 4 | 月２回 |
| 3 | 月１回 |
| 2 | しばらくいい |
| 1 | リピなし |

旧 `nextTiming` がある場合は、「明日」「来週」「2週間」「月1」「しばらく」「なし」などの語から周期を推定します。

## 10. 抽出ロジック

### 材料抽出

キャプションから以下のような表現を拾います。

- `鮭 3切れ`
- `しめじ 1袋`
- `玉ねぎ 1個`
- `しょうゆ 大さじ1`
- `めんつゆ 適量`

抽出後、材料カテゴリを簡易判定します。

- 野菜
- 肉
- 魚
- 卵・乳製品
- 大豆・加工品
- 主食
- 缶詰
- 調味料
- その他

### 調理方法抽出

キャプション内の文章から、調理に関係する動詞を含む文を手順として抽出します。

対象例:

- 切る
- 炒める
- 焼く
- 煮る
- 蒸す
- 混ぜる
- のせる
- かける
- 包む
- 加える
- ゆでる
- 冷やす
- 仕上げる

番号付き手順がある場合は、番号を外して最大6件まで表示します。

## 11. 現状の制約

- バックエンドなし
- ログインなし
- ユーザーアカウントなし
- 端末間の自動同期なし（JSON書き出し／読み込みによる手動移行は可能）
- 写真はサーバーへアップロードせず、リサイズして端末内（localStorage）に保存
- SNSキャプションの実取得なし
- データは端末ごとのブラウザ保存
- 保存・記録の日付は端末の現在日付を使用（既存デモデータの日付は固定）

GitHub Pagesで動くこと:

- アプリ画面表示
- URL入力
- URL種別判定
- キャプション貼り付け
- 材料・調理方法の簡易抽出
- レシピ保存
- リピ周期保存
- 献立提案
- レシピ編集・削除
- リピ記録削除
- 料理写真の登録
- 家族メンバー編集
- JSON書き出し／読み込み

GitHub Pagesだけでは動かないこと:

- SNS URLからの本文自動取得
- ログイン
- 端末間同期
- サーバー保存

## 12. 将来拡張候補

優先度高:

- バックエンド/API連携によるSNS本文取得
- 週単位の献立編集
- 買い物リスト
- 端末間同期

優先度中:

- 栄養・食材バランスの可視化
- 料理写真カレンダー
- 家族別の苦手食材メモ
- レシピタグの手動編集

優先度低:

- 外部レシピサービス連携
- 詳細な栄養アドバイス
- 複数家庭・共有グループ対応

- SNSレシピ動画を保存できる
- 作った後に家族別リピ周期を残せる
- 「今週何を作るか」の候補が見える
