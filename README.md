# リピごち ![Tier](https://img.shields.io/badge/tier-hobby-blue)

> 家族の「また食べたい周期」を残す、ごはんメモ

## これは何？

リピごちは、SNSで見つけたレシピ動画を保存して、作ったら家族ごとのリピート周期を記録する静的MVPです。
「明日でも」「毎週」「月２回」などの周期から、今週の献立候補を軽く整えます。

## こんな時に使えます

- SNSで見つけたレシピ動画をあとから探し直したいとき
- 家族ごとの「また食べたい」ペースを残したいとき
- 忙しい週の献立候補を、保存済みレシピからさっと決めたいとき

## 使い方

- デモ: https://165cm.github.io/matagochi/
- 初回は「空ではじめる」か「サンプルを見る」を選びます。
- 中央の「登録」からショート動画リンクを入力し、URLから取得または手動入力でレシピを保存します。
- 「コレクション」画面では、保存済みレシピを検索・編集・削除できます。
- 作ったレシピは「リピ周期」画面で、家族ごとに6段階の周期とメモ、料理写真を記録できます。
- 「献立」画面では、直近の調理日とリピ周期から今週の候補を確認できます。
- 「設定」画面で、材料の表示人数、家族メンバーの編集、JSONバックアップの書き出し／読み込み、レシピとリピ記録の全件削除ができます。

## できること

- Instagram / Facebook / TikTok / YouTube Shorts のURL判定
- Cloud Run API設定済み環境でのYouTube説明文取得
- Gemini FlashによるYouTube説明文からの材料メモ作成
- キャプション貼り付けからの材料・調理方法の抽出
- 1人前で保存した材料メモの人数分表示
- 保存済みレシピの検索・編集・削除
- 家族別のリピ周期記録
- リピ周期にもとづく1週間の献立提案
- 料理写真の登録（任意）とリピ記録履歴の削除
- 家族メンバーの追加・名称変更・削除
- JSONファイルによる手動データ連携
- 設定画面からのレシピ・リピ記録の全件削除

データはこの端末にのみ保存されます。端末を変えるときは、JSONバックアップの書き出し／読み込みで移せます。

YouTube連携は動画の説明文を使います。字幕トラックの取得や、ログイン付きの自動同期はできません。
Cloud Run APIを設定していない環境では、これまで通りキャプションを貼り付けて材料メモを作れます。

## 開発メモ

YouTube説明文取得APIは `api/` にあります。Cloud Runで動かす場合は `api/.env.example` を参考に、YouTube Data APIキーとGoogle Cloud / Vertex AIの設定を環境変数で渡します。
フロント側では `window.MATAGOCHI_API_BASE_URL` にCloud RunのURLを設定すると「URLから取得」が有効になります。
